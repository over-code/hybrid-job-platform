# Overview

## 1) Что это за проект
Гибридная платформа (учебный прототип) для демонстрации вёрстки и базовой интерактивности:
- вакансии (job board)
- фриланс‑проекты

## 2) Ограничения (Scope)
- Только фронтенд: **без сервера** и **без базы данных**.
- Данные и “сессия” пользователя хранятся в `localStorage`.
- Никаких реальных email‑отправок, пушей, интеграций.

## 3) MVP — что обязательно должно быть
### Экраны (минимальный набор)
- Главная (лендинг)
- Вход
- Регистрация
- Восстановление пароля (учебная заглушка)
- Сброс пароля (учебная заглушка)
- Каталог вакансий
- Детали вакансии
- Каталог проектов
- Детали проекта
- Профиль пользователя
- О нас
- Контакты

### MVP-цели качества (Definition of Done)
- Проект запускается через Live Server.
- SPA-навигация работает без перезагрузки.
- Данные рендерятся из `localStorage` по модели `DATA_MODEL.md`.
- Сценарии из `QA.md` проходят.

## 4) Текущий статус (v1.0.1)

### Выполнено
- [x] Полноценная визуальная составляющая проекта (основной UI/UX)
- [x] Доведены до целостного вида основные экраны и паттерны взаимодействия (SPA + localStorage)
- [x] Профиль (как часть UI): расширенные настройки кандидата + мини-сводка
- [x] Документация актуализирована

### Примечание по истории
- `v1.0.0` — техническая бета: каркас/архитектура/страницы без полноценного дизайна.

### Что НЕ входит в текущую версию (Future)
- Реальная безопасность (хэширование паролей, токены, 2FA)
- Email/push уведомления
- Чаты, обмен сообщениями между пользователями
- Реальный бекенд/API

## 5) Дальнейшее развитие

Если проект будет продолжать развиваться, рекомендуется рассмотреть:
1. Реализацию полноценного бэкенда
2. Добавление системы уведомлений
3. Внедрение системы рейтингов и отзывов
4. Расширение функционала личного кабинета
